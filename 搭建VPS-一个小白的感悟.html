一.关于VPS

1.什么是VPS

 VPS :简单点说可以理解为服务器.

 当一台服务器用软件分割开以后，就成了多台服务器，它们有独立的操作系统，具有独立的IP，这个时候每一个小的独立操作系统，就是一个VPS。

 还是不明白的话,VMstation虚拟机了解吧?在上面部署一个系统作为服务器,而VPS就是你部署在了国外.

## 二.搭建SS
 购买成功,但不能马上翻墙,需要设置(搭建)SS(shadowsocks-server)和下载ss客户端.
### 1.搭建SS
 依次点击My services、KiwiVM Control Panel进入控制面板，VPS初始系统是Centos7_x86_brr,默认的系统即可安装SS；

 现在大部分文章写的都是点击Shadowsocks Server、Install Shadowsocks Server进行一键安装;现在已经取消一件安装,可以在install shadowsocks进行一键安装.

 如图:点击安装




 安装好后点击go back



 回到Shadowsocks server control面板



 以上是通过一键安装进行SS-server搭建的;还可以通过代码进行安装,可以参考:https://teddysun.com/357.htm.

 我在一键安装后了”500 502 504”等错误提示,所以尝试了代码命令安装,最后发现问题跟怎么安装没关系,所以不详细描述了.
完成ss-server部署,下面是客户端下载


### 2.ss-client下载
 上面截图已经给出了不同平台客户端下载链接,在安装客户端前,先让大家了解下容易混淆的地方.

 登录KiwiVM主控制板的时候需要输入IP和密码



 在你安装好ss-server,点击”go back”时,面板也有一个密码,注意:这个端口号和密码才是你在ss-client端配置需要的.



 还有一个root账号密码,以root账号运行时需要,初始会邮件发给你,每次reload系统时会生成新的.




 根据需要,下载不同平台的客户端,然后配置ss-client,按照上面说的填到对应的框里,注意:密码填对;点击确定




 设置代理模式为PAC模式

 PAC模式:访问PAC文件内的网址,才用VPS;

 全局模式:访问国内网站也走VPS



完成以上配置,幸运的话你就可以访问墙外的资源了

不过总会有遇到问题的人,比如我

三.SS问题解决

我自己在Mac上和win7,win10上做了尝试,存在最多问题的是Windows系统,而且我用的比较多的也是Windows,所以问题侧重Windows系统

我遇到的问题

 主要有:

 ”500 Internal Privoxy Error”




 ”502 server dropped connection”




 ”504 Connect to to www.facebook.com:80 fai failed: SOCKS protocol error”



解决方法

 以上问题主要是你部署在VPS的服务器防火墙没有关闭,还有就是你下载的ss-client有问题,以我部署部署系统(centos6-x86)为例:

 关闭centos6防火墙命令:

/sbin/iptables -A INPUT -m state --state NEW -m tcp -p tcp --dport SERVER_PORT_NUMBER -j ACCEPT`

/etc/init.d/iptables save

/etc/init.d/iptables restart

 执行命令后,国外服务器的防火墙已经关闭,还有问题就是ss-client的问题,而Windows的ss-client是最容易出现问题的,下面提供我win7,win10ss-client的客户端下载
